# Accessibility (A11Y) checklist (Stage 12)

This project aims to keep UI components accessible by default.

## Quick checklist

- All inputs have visible labels (or `aria-label`).
- Buttons have clear text; icon-only buttons have `aria-label`.
- Dialogs:
  - Close on Escape
  - Overlay click closes
  - Focus outline visible (`focus-visible:ring`)
- Color is not the only signal for status (use text + badge label).
- Tables have headers (`<th>`).
- Links are identifiable (underline on hover at minimum).

## Known gaps

- Our custom `Dialog` does not trap focus yet.
- No keyboard-based reordering for the step editor yet (up/down buttons are clickable).

Planned improvements:
- Focus trap in Dialog
- Add `aria-live` to success/error banners where appropriate
